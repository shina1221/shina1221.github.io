<br>   
신규 고객을 더 늘리고 싶다 -> 신규 고객을 지금 늘리는게 중요한가?  
얼마정도 필요해? -> 비용대비 효과는 어느정도를 기대할 수 있어?  
그럼 잘 돼? -> 비용을 지불하면 효과는 있겠는데 성공이랑 실패 기준은 뭔데?  
데이터에 기반한 의사결정이 필요하다.  
<br>
데이터 기반 의사결정(Data Driven Decision Ma king)  
1. 정확한 정보제공으로 오류와 실수를 줄인다.  
2. 실시간으로 데이터를 수집해 시장동향, 고객 행태등을 즉각 대응할 수 있다.  
3. 문제를 사전에 식별해 리스크를 최소화 할 수 있다.  
4. 데이터 기반 분석으로 경쟁우위를 확보할 수 있다.  
<br>
ex)  
마케팅팀  
- 고객의 구매이력, 선호도, 행동 패턴 등을 분석해 세분화된 고객 세그먼트 식별  
- 어떤 고객이 특정 제품에 더 관심을 가지는지, 어떤 마케팅 채널이 효과적인지 등을 파악->마케팅 예산을 효율적으로 할당  
<br>
데이터 드리븐 보고 전략  
목표(가설)설정 > 데이터 수집 > 데이터 정제 > 데이터 분석,EDA,모델링 > 평가/해석 > 보고 프로세스  
  
**목표(가설)설정**  
- 어떤 문제를 해결하고자 하는가?  
ex)온라인 쇼핑몰 매출을 증대시킨다. / 마케팅을 어떻게 해야할까?  
- 분석의 목적은 뭐지?  
ex)그럼 온라인 쇼핑몰 매출에 영향을 미치는 요인은 뭐지? / 고객의 구매 패턴을 이해해서 타겟 마케팅을 진행해본다.   
- 필요한 결과는 뭐지?  
ex) - / 고객 세그먼테이션 및 선호도 예측 모델이 결과물로 나온다.  
<br>

**데이터 수집**  
어떤 데이터를 수집하지? 
- 서비스 목적과 비즈니스요구, 분석 목표에 맞는 필요데이터 정의  
데이터를 어떻게 연결하지?  
- 데이터가 여러 테이블에 나눠 저장된 경우 각 테이블을 연결하기 위한 식별 키 필요  
어떤 모양으로 저장하지?  
- 텍스트? csv? json?  
데이터 수집주기는?  
- 데이터의 변동성과 업데이트의 필요성을 고려해 얼마나 자주 데이터를 업데이트 할 건지 결졍해 데이터의 정확성을 보장  
<br>

**데이터 정제**  
이상치 제거:일반적인 패턴에서 벗어난 극단적인 값 제거  
표준화:데이터의 형식이나 단위를 표준화해 데이터 일관성을 유지하고 분석 과정 수행 효율을 향상시킴  
결측치 처리: 삭제나 대체로 적절히 처리하지 않을 경우 분석 결과에 왜곡이 일어남    
데이터 형식 통일  
샘플링:데이터 크기 축소로 데이터 분석에 필요한 자원이나 시간 절약 가능  

**데이터 분석**
*EDA(Exploratory Data Analysis 이하 탐색적 데이터 분석)  
데이터를 탐색하고 이해하기 위한 과정으로 데이터가 어떻게 되어있는지 파악하고 패턴이나 관계를 발견하는 과정  

데이터 탐색  
데이터 구조, 변수유형, 분포 등 탐색  
데이터셋 크기, 변수 갯수, 형식 등 파악  
  
변수간 관계 탐색  
상관관계, 연관성 분석  
ex)산점도, 상관행렬, 히트맵 등  
데이터의 특성 파악  
  
데이터 분석  
패턴 분석/트렌드 발견 등  
세계열 변화, 클러스터링 등   
<br>  

**분석 평가/해석**
1. 분석 결과해석  
데이터 분석과 통계적 결과 해석  
ex) 마케팅 성과평과: 클릭률, 전환율지표 해석  
2. 결과 평가  
분석 결과의 타당성, 유의성, 신뢰성 평가  
ex)통계의 유의성 검정, 모델의 성능평가(roc 곡선 아랜 면적인 ACU로 )  
  
*타탕성: Validity - "정확한 것을 측정하는가?"  
정의: 측정도구가 측정하고자 하는 개념이나 의도한 목적을 얼마나 정확하게 측정하고 있는지를 나타내는 정도  
ex)영어 실력을 평가하는 시험이 독해/문법/청해를 다루지 않고 지식 상식만 묻는다면 타당성이 낮은 것임  
종류: 내용 타당성기준 타당성구성 타당성 등  
    
*유의성: Significance - "우연이 아닌 의미 있는 결과인가?"  
정의: 통계적 가설 검정에서 나온 결과가 우연히 발생한 것이 아니라데이터 내에서 실제적인 차이나 영향력을 가지고 있는지를 확률적으로 나타내는 것  
핵심: 통계적 유의성(p-value)을 통해 판단 (보통 0.05 이하일 때 유의하다고 봄)  
내용: '차이가 있는가/없는가''영향이 있는가/없는가'를 판단하는 기준  
  
*신뢰성: Reliability - "일관되게 측정하는가?"  
정의: 동일한 대상에게 동일한 측정 도구를 반복적으로 사용했을 때일관성 있고 안정적인 결과를 도출하는 정도  
핵심: '일관성''안정성''예측 가능성'  
예시: 몸무게를 3번 쟀는데 60kg, 60kg, 60kg이 나오면 신뢰성이 높은 것. (단 실제 몸무게가 70kg이라면 타당성은 낮음)  
측정법: 재검사법반분법 등  

3. 인사이트 도출  
데이터에서 발편된 패턴이나 관계를 이해하고 이를 바탕으로 의사결정을 내림  
4. 의사결정지원  
분석 결과해석 > 의사결정권자에게 전달 > 계획수립 > 비즈니스 전략 개선, 문제 해결 등  
<br>  

데이터 드리븐 보고  
간결하게 요약한 상태로 시작  
빠르고 쉽게 이해할 수 있도록 간단하고 명확하게 설명  
복잡한 분석이나 통계적인 용어는 지양  
그래픽 요소를 활용해 내용전달  
데이터를 어떻게 해석했는지 어떤 인사이트를 얻을 수 있었는지 표현  
<br>  
데이터 드리븐 보고유형  
시각화: 패턴이나 관계 파악용도  
선형회귀분석: 종속변수와 독립변수 간 관계 분석  
가설검정: 통계적 가설을 검정해 데이터 분석 결과의 유의성을 평가하는 과정  
기초통계  
모델링: 패턴을 학습하고 예측 수행하는 수학적 모델 구축  
예측: 모델링을 통해 학습된 모델을 사용해 미래의 결과 예측
