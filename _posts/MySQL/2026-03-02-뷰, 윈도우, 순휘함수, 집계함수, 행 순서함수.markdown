뷰: 다른 테이블에서 파생된 데이블로 물리적으로 데이터를 저장하지 않고 논리적으로만 데이터가 존재해  
뷰를 사용한 질의 시에는 DBMS에서 뷰 정의에 따라 질의를 재작성해 수행  
뷰 정의에 따라 질의하고 다른 테이블에서 파생었기 떄문에 빅쿼리를 껐다가 다시 켜서할 땐 뷰에 대한 쿼리를 다시 실행해줘야 함  
  
뷰  
독립성 테이블 구조가 변경되어도 뷰를 사용하고 있는 응용프로그램은 변경하지 않아도 됨  
편리성 자주 사용되는 복잡한 쿼리를 미리 뷰로 정의 해 놓으면, 추후 쿼리는 간단한 형태로 표현할 수 있음  
보안성  
사용자의 권한에 따라 열람 가능한 데이터를 다르게 할 수 있음  
권한에 따라 확인 가능한 컬럼을 정의하여 뷰를 생성하면, 기본 테이블 노출 없이 접근 제어 가능  
ex)구독 회원제 서비스가 없던 플랫폼이 이후 이런 서비스를 만들고 싶을 때 회원 여부 컬럼을 만들어야 할 것임  
테이블 구조가 변경되어야 하기에 이 테이블에서 데이터를 가지고 있던 모든 프로그램에 일괄적으로 대응해야 한다면  번거롭고 일괄 대응을 못할시 에러가 뜰텐데, 이 때 응용프로그램이 뷰를 바라보게 하면 서비스가 안정적으로 운영 가능   
편리성  
여러 함수를 적용해둔 복잡한 쿼리들이 있을 때 결과물을 한번 뷰에 저장하고 
나머지 분석물은 파생된 테이블에 기반해서 이어서 작업하면 됨  
보안성  
뷰를 생성하면 사용자 권한에 따라 확인 가능한 컬럼이 차이가 남


집합 연산자  
두 개 이상의 테이블에서 JOIN을 사용하지 않고 연관된 데이터를 조회하는 방법 중 하나  
테이블에서 SELECT한 컬럼의 수와 각 컬럼의 데이터 타입이 테이블 간 동일해야 함  

Union은 두 개의 테이블을 하나로 만드는 연산으로 합친 이후에는 중복된 데이터를 제거  
```
SELECT UserID, ItemID FROM Table1
UNION
SELECT UserID, ItemID FROM Table2;
```  
**빅쿼리는 Union 사용불가//UNION은 사용가능하지만 중복제거를 하지 않음**

MS SQL: INTERSECT 연산 가능  
MySQL: INTERSECT 연산이 불가하여 INNER JOIN 등을 사용해야 함  
```
SELECT UserID, ItemID FROM Table1
UNION #INTERSECT  
SELECT UserID, ItemID FROM Table2;  
```

윈도우 함수(Window Function)  
순위, 집계 등 행과 행 사이의 관계를 정의하는 함수로 OVER 구문을 필수로 사용함  
  
윈도우 함수(Window Function) 구조 살펴보기  
```
SELECT WINDOW_FUNCTION(ARGUMENTS)
OVER ([PARTITION BY 절] [ORDER BY 절] [WINDOWING 절])
FROM 테이블명;
```  
구조 설명  
ARGUMENTS 윈도우 함수에 따라서 필요한 파라미터  
PARTITION BY 전체 집합에 대해 소그룹으로 나누는 기준  
ORDER BY 소그룹에 대한 정렬 기준  
WINDOWING 행에 대한 범위 기준  
```
윈도우 함수(Window Function) 구조 살펴보기
SELECT RANK( ) OVER (ORDER BY Profit DESC)
FROM 테이블명;

SELECT SUM(Sales) OVER (PARTITION BY Country)
FROM 테이블명;

-- 나라별 sales 순위 보기   
select country, sum(sales) over (partition by country)
from pj-1-489007.part1.superstore;

-- 구매자별 sales 순위 보기
select CustomerID, Profit, RANK() OVER (ORDER BY Profit DESC)
from pj-1-489007.part1.superstore;
```  

순위 함수 (RANKING FUNCTIONS)  
```
SELECT RANK( ) OVER ([PARTITION BY 절] [ORDER BY 절] [WINDOWING 절])
```  
함수 설명  
RANK( ) 동일한 값에는 동일한 순위를 부여  
DENSE_RANK( ) RANK와 동일하나, 동일한 순위는 한 건으로 취급  
ROW_NUMBER( ) 동일한 값이라도 고유한 순위를 부여  

```
SELECT
Category,
SubCategory,
Sales,
RANK() OVER (PARTITION BY Category ORDER BY Sales DESC) AS Rank,
DENSE_RANK() OVER (PARTITION BY Category ORDER BY Sales DESC) AS DenseRank,
ROW_NUMBER() OVER (PARTITION BY Category ORDER BY Sales DESC) AS RowNum
FROM pj-1-489007.part1.superstore;

-- 카테고리별 sales가 가장 잘나온 서브카테고리의 sales 확인하기
SELECT
  Category,
  SubCategory,
  total_sales
FROM (
  SELECT
    Category,
    SubCategory,
    SUM(Sales) AS total_sales,
    RANK() OVER (PARTITION BY Category ORDER BY SUM(Sales) DESC) AS rnk
  FROM `pj-1-489007.part1.superstore`
  GROUP BY Category, SubCategory
)
WHERE rnk = 1; -- limit 1로 참조하지 않음  
-- 윈도우 함수는 group by 이후에 실행  

-- 카테고리 별 가장 큰 매출이 발생한 단일 주문 레코드 확인하기 
SELECT
  Category,
  SubCategory,
  Sales,
  RANK() OVER (PARTITION BY Category ORDER BY Sales DESC) AS rnk
FROM `pj-1-489007.part1.superstore`
QUALIFY rnk = 1;
-- QUALIFY = 윈도우 함수 계산 결과를 WHERE처럼 걸러주는 절
-- where는 윈도우 함수보다 먼저 실행되기에 그 결과를 쓰지 못해 qualify 사용
```  
  
집계함수  
```
SELECT SUM(col1) OVER ([PARTITION BY 절] [ORDER BY 절] [WINDOWING 절])
```  
함수 설명  
SUM(컬럼) 컬럼을 기준으로 합계를 계산  
AVG(컬럼) 컬럼을 기준으로 평균을 계산  
MAX(컬럼) 컬럼을 기준으로 최대값을 계산  
MIN(컬럼) 컬럼을 기준으로 최소값을 계산  
  
윈도잉 절  
```
SELECT RANK( ) OVER ([PARTITION BY 절] [ORDER BY 절] [WINDOWING 절])

SELECT SUM(Sales) OVER (PARTITION BY Category
ROWS BETWEEN UNBOUNDED FOLLOWING AND CURRENT ROW)
```
함수 설명  
CURRENT ROW 윈도우의 시작 위치 (현재 행)  
UNBOUNDED PRECEDING 윈도우 시작 위치가 첫번째 행  
UNBOUNDED FOLLOWING 윈도우 마지막 위치가 마지막 행  
```
-- 고객별 매출 확인
SELECT
OrderID,
Sales,
SUM(Sales) OVER (PARTITION BY OrderID) AS TotalSalesByOrder
FROM `pj-1-489007.part1.superstore`

-- Windowing 절 실습
-- 같은 날짜 안에서 주문 순서대로 매출을 계속 더해가는 누적합 계산
SELECT
OrderDate,
OrderID,
Sales,
SUM(Sales) OVER (
PARTITION BY OrderDate
ORDER BY OrderID
-- ROWS BETWEEN A and B 구조
ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW -- 맨 처음 행부터 현재 행까지 모두 더하기
) AS CumulativeSales
FROM `pj-1-489007.part1.superstore`
ORDER BY OrderDate, OrderID;


#mysql 버전
SELECT *
FROM (
    SELECT 
        OrderDate,
        CumulativeSales,
        RANK() OVER (
            PARTITION BY OrderDate 
            ORDER BY CumulativeSales DESC
        ) AS rnk
    FROM (
        SELECT
            OrderDate,
            OrderID,
            Sales,
            SUM(Sales) OVER (
                PARTITION BY OrderDate
                ORDER BY OrderID
                ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
            ) AS CumulativeSales
        FROM superstore
    ) AS t1
) AS t2
WHERE rnk = 1;

#날짜별 마지막 행 뽑기
SELECT *
FROM (
    SELECT 
        OrderDate,
        OrderID,
        SUM(Sales) OVER (
            PARTITION BY OrderDate
            ORDER BY OrderID
        ) AS CumulativeSales,
        ROW_NUMBER() OVER (
            PARTITION BY OrderDate
            ORDER BY OrderID DESC
        ) AS rn
    FROM superstore
) t
WHERE rn = 1;
```  
  
행 순서 함수  
```
SELECT FIRST_VALUE(col1) OVER (…)
```  
함수 설명  
FIRST_VALUE(컬럼) 가장 먼저 나온 값을 구함  
LAST_VALUE(컬럼) 가장 나중에 나온 값을 구함  
LAG(컬럼, n) 이전 n번째 행을 가져옴  
LEAD(컬럼, n) 이후 n번째 행을 가져옴  
  
```
-- 행 순서 함수 (VALUE ACCESS FUNCTIONS) 실습
SELECT CustomerID, OrderDate, Sales,
FIRST_VALUE(Sales) OVER (
PARTITION BY CustomerID
ORDER BY OrderDate
) AS FirstSale
FROM
`ProjectID.DatasetID.superstore`
ORDER BY CustomerID, OrderDate, FirstSale;

-- 행 순서 함수 (VALUE ACCESS FUNCTIONS) 실습
SELECT CustomerID, OrderDate, Sales,
LAST_VALUE(Sales) OVER (
PARTITION BY CustomerID
ORDER BY OrderDate
) AS FirstSale
FROM
`ProjectID.DatasetID.superstore`
ORDER BY CustomerID, OrderDate, FirstSale;

-- 행 순서 함수 (VALUE ACCESS FUNCTIONS) 실습
SELECT CustomerID, OrderDate, Sales,
LAG(Sales, 1) OVER (
PARTITION BY CustomerID
ORDER BY OrderDate
) AS PreviousSale
FROM
`ProjectID.DatasetID.superstore`
ORDER BY CustomerID, OrderDate, FirstSale;

-- 행 순서 함수 (VALUE ACCESS FUNCTIONS) 실습
SELECT CustomerID, OrderDate, Sales,
LEAD(Sales, 1) OVER (
PARTITION BY CustomerID
ORDER BY OrderDate
) AS NextSale
FROM
`ProjectID.DatasetID.superstore`
ORDER BY CustomerID, OrderDate, FirstSale;
```
