추론과 예측  

추론  
- 모집단에 대한 통찰을 얻기 위함
- 가설 검정을 통해 특정 주장이나 이론을 검증하기 위함  
- 데이터가 특정 패턴을 따르는지 확인하기 위함  
방법론  
기술통계: 데이터의 요약과 설명(평균, 중위수, 분산 등)  
추정: 모집단의 파라미터를 샘플 데이터로부터 추정(평균, 비율, 표준편차 등)  
가설검정: 특정 가설이 데이터에 의해 지지되는지 검정(t검정, 카이제곱 검정 등)  
회귀분석: 변수간 관계 설명
<br>

예측  
- 잠재적 위험이나 기회를 사전에 인지하기 위함  
방법론  
회귀분석: 연속형 변수 예측  
분류:이산형 변수 예측  
시계열 분석: 시간에 따른 값 예측  
머신러닝: 다양한 데이터 패턴 학습 후 예측

확률과 분포  
확률  
- 어떤 사건이 발생할 가능성으로 0애서 1사이의 값을 가짐  
사건:특정 결과  
ex) 사용자가 웹 사이트에 방문하는 것   
표본공간:모든 가능한 결과의 집합  
ex)웹사이트 방문, 미방문  
확률값: 특정 사건이 발생할 가능성  
  
고전적 확률  
모든 결과가 동일한 가능성을 가지는 경우로 간주  
ex)AB test의 경우 A안과 B안이 선택될 확률이 동일하다고 가정 후 테스트  
  
확률의 활용사례    
사용자 행동분석
특정 사용자 행동의 발생 확률을 계산해 서비스 개선에 활용  
ex) 사용자가 특정 페이지를 방문한 후 구매할 확률이 10%라면 이 확률을 높이기 위해 구매 유도 전략 개발  
  
AB test  
두가지 이상의 옵션 중 어느것이 더 효과적인지 평가  
ex)마케팅에서 A안과 B안 중 어느 글귀가 더 클릭될 확률이 높은지 비교해 클릭 확률이 높은 제목을 캠페인에 활용  
  
기능 사용 예측  
새로운 기능의 성공 가능성 예측  
ex) 새로운 채팅 기능을 출시하기 전에 비슷한 기능의 과거 사용 데이터를 분석해 새로운 기능이 사용될 확률 예측 후 개발 자원을 효율적으로 배분

분포  
데이터가 특정 값이나 구간에 어떻게 흩어져 있는지를 나타냄  
기대값: 확률분포의 평균값으로 장기적으로 기대할 수 있는 값  
분산: 데이터가 평균값을 기준으로 얼마나 퍼져 있는지를 나타냄  
표준편차: 분산의 제곱근  
왜도: 분포의 비대칭성으로 양의 왜도는 오른쪽 꼬리가 긴 분포, 음의 왜도는 왼쪽 꼬리가 긴 분포  
첨도: 분포의 꼬리 부분의 두터움을 의미하며 높은 첨도는 꼬리가 두터움  
  
**사용자 행동분석에서의 분포 왜곡 활용**  
사용자가 웹 사이트에서 소비하는 시간이 양의 왜곡을 보일 경우 대부분이 짧은 시간을 보내고, 일부는 긴 시간을 보낸다는 것을 의미  
사용자가 오랜 시간을 보내는 페이지나 기능을 분석해 다른 사용자에게도 확산할 방안을 모색하고 평균 사용시간을 증가시키기 위해 특정 콘텐츠나 기능을 강화  
  
웹사이트 가입 절차에서 이탈률이 음의 왜곡을 보인다면, 대부분의 사용자가 가입 절차를 쉽게 완료하지만 일부는 초기단계에서 이탈한다는 것을 의미  
가입 절차를 간소화하거나 초기 단계에서 사용자가 겪는 불편함을 줄이기 위한 UX 개선 작업  
  
**마케팅 전략 수립에서의 분포 왜곡 활용**  
마케팅 캠페인의 반응률이 양의 왜곡을 보인다면 대부분의 사용자가 낮은 반응을 보이지만 일부는 매우 높은 반응을 보인다는 것을 의미  
높은 반응을 보이는 사용자 그룹을 분석해 이들의 특성을 파악하고 비슷한 특성을 가진 잠재 고객을 대상으로 맞춤형 마케팅 전략 수립  

특정 제품의 구매빈도가 음의 왜곡을 보인다면 대부분의 사용자가 자주 구매하지만 일부는 드물게 구매한다는 것을 의미  
드물게 구매하는 사용자들을 대상으로 재구매를 유도하는 프로모션 시행

상관관계와 인과관계  
인과관계: 한 변수가 다른 변수에 직접적인 영향을 미치는 원인과 결과의 관계  
산점도를 그려봤을 때 음의 상관관계나 양의 상관관계가 명확하게 나오지 않는다면 다른 인과관계가 있거나 다른 상관관계를 가지는 요인들이 있을 수 있다고 볼 수 있음  
ex)  
상관관계)광고지출이 높으면 매출이 늘어날 수 있다.  
인과관계)매출증가는 시즌별 수요증가, 새로운 제품 출시, 경제 상황등의 여러 요인에 영향을 받을 수 있다.  
    
**서비스 기획에서의 상관관계, 인과관계 활용**  
상관관계 활용  
특정 페이지 방문과 구매전환율 간의 상관관계를 분석해 어떤 페이지가 구매를 유도하는지 파악  
특정 기능의 사용 빈도와 사용자 만족도 간 상관관계를 분석해 중요한 기능을 개선하거나 추가 가능  

인과관계 활용  
AB테스트로 새로운 기능이 사용자 유지율에 미치는 영향을 평가해 기능 개선 여부 결정  
특정 마케팅 활동이 실제 판매 증가로 이어지는지 확인해 마케팅 전략 수립

조건부 확률  
어떤 사건 A가 발생한 조건하에서 다른 사건 B가 발생할 확률  
ex)쿠폰을 뿌렸는데 이게 구매에 효과가 있었을까?  
        쿠폰받음  쿠폰받지않음  총  
구매      300         150      450  
비구매    100         450      550  
총        400         600     1000  

쿠폰을 받은 사람이 구매할 확률 P(B|A)
P(A|B)=P(A교집합B)/P(B)  
P(A교집합B):쿠폰을 받고 구매한 확률  
P(A):쿠폰을 받은 확률  
  
P(A교집합B)=300/1000 = 0.3  
P(A)=400/1000 = 0.4  
P(B|A)=0.3/0.4 = 0.75(쿠폰을 받은 사람이 구매할 확률 75%)  
  
쿠폰을 받지 않은 사람이 구매할 확률 P(B|not A)
P(B|not A)=P(not A교집합B)/P(not A)  
P(not A교집합B):쿠폰을 안 받고 구매한 확률  
P(not A):쿠폰을 안 받은 확률  
  
P(not A교집합B)=150/1000 = 0.15  
P(not A)=600/1000 = 0.6  
P(B|not A)=0.15/0.6 = 0.25(쿠폰을 안 받은 사람이 구매할 확률 25%)  
  
따라서 쿠폰을 받은 고객의 구매 확률은 75%  
쿠폼을 받지 않은 고객의 구매확률 25% 보다 훨씬 높다. -> 쿠폰을 뿌린게 유의미한 행동이었다.   
  
**팁!**
하지만 쿠폰을 뿌린 시점에서 비용이 나갔다는 의미기 때문에 쿠폰을 뿌리고도 수익을 가져오는가에 대한  
최적화에 대한 계산이나 데이터 분석 등을 통해 비용대비 수익 기대효과 확률을 키울 수 있는 방향으로 전략을 수립하는것 뿐만 아니라  
쿠폰 뿌렸을 대 신규 고객이 늘어날 수도 있고, 재구매가 늘어나서 충성 고객이 늘어날 수도 있고  
충성고객이 늘어나면서 해당 고객에 대한 수익이 더 높아질 수도 있기 때문에 다양하게 고민해보는 것을 추천  
어떤 고객을 타겟팅 할 것인가를 계산할 때 조건부 확률을 활용 가능

AB 테스트  
**두가지 이상**의 변형을 비교해 어떤 변형이 더 나은 성과를 내는지 평가  
변형 A는 일반적으로 현재상태(컨트롤)이고, 변형B는 변경된 상태(실험) 의미  
AB 테스트의 주요 단계  
목표설정 > 가설설정 > 변형 디자인 > 표본분할 > 실험수행 > 데이터 분석(기초통계, 조건부확률을 통한 해석 등) > 결론 도출  
*표본분할:사용자를 무작위로 두 그룹으로 나눔 ex)나이, 지역,나라, 성별에 대한 편향이 있어선 안됨  
<br>  
데이터 기반 의사결정으로 서비스 기획자가 나라면 이렇게 할텐데 해서 진행했는데 생각보다 다수의 사용자들이 그렇게 행동하지 않았을때  
사용자 중심 접근성이 떨어지는 경우가 생길 수 있다. AB test를 하면 작은 그룹에서 먼저 평가를 하기 때문에 이런 위험 최소화 가능  
*AB 테스트는 내가 운영하는 서비스의 모든 이용자를 대상으로 AB를 나누는 것이 아님  
여기서 일부만 랜덤으로 추출해 일부 사용자들에게 랜덤하게 기능을 노출시켜 다수에 대한 방향 수립가능   
*또한 AB테스트를 하면서 본인의 의견을 위해 일부 실험을 비트는 경우가 있는데 AB 테스트는 고객의 행동을 객관적으로 파악해 더 큰 가치를 주기 위함이기에 이런 것은 지양  
<br> 
충분한 표본 크기로 실험하되, 
일시적인 변동이 있을 수 있고, 지속적인 평가를 해야하는데 시간마다, 월마다, 계절마다 등등 이용자들의 특성이 달라질 수 있다.  
계절성을 반드시 반영해야하는 수준으로 해야한다는 것은 아니고, 유의미한 데이터를 얻을 수있는 수준으로 진행해도 충분하다.  
외부요인을 통제해 
실험기간동안은 특정 프로모션이 있거나 시즌 변화가 있는 등 다른 요인이 생기면 결과에 영향을 미치기 때문에 A와B는 동일한 환경에서 테스트를 진행

**활용 방안**  
랜딩 페이지 콘텐츠  
목적: 랜딩 페이지의 전환율 개선  
A버전: 기존 컨텐츠와 레이아웃  
B버전: 새로운 콘텐츠와 단순화된 레이아웃  
특정항목: 전환율, 방문자 당 평균 체류 시간, 이탈률  
  
고객 온보딩 프로세스  
목적: 고객 온보딩 과정을 개선해 사용자 활성화  
A버전: 기존 온보딩 프로세스  
B버전: 단계별 가이드와 도움말이 추가된 온보딩 프로세스  
측정 항목: 온보딩 완료율, 활성 사용자 비율, 초기 사용자 이탈률  
ex) 온보딩 프로세스 회원가입 페이지 끝나자 마자  
알림을 줄까? 쿠폰을 쏠까? 가이드 메뉴얼을 둘까? 체험으로 바로 시작하게 할까 이런 고민을 할때 AB 테스트를 활용해볼 수 있음  

모바일 앱 내 기능 테스트  
목적: 새로운 기능이 사용자 참여도에 미치는 영향파악  
A버전: 기존 기능만 포함된 앱  
B버전: 새로운 기능이 추가된 앱  
측정항목: 기능 사용률, 앱 유지율, 사용자 피드백  
  
소셜 미디어 포스트 형식  
목적: 소셜 미디어 포스트의 참여도 향상  
A버전: 텍스트 중심의 포스트  
B버전: 이미지와 동영상이 포함된 포스트  
측정 항목: 좋아요 수, 댓글 수, 공유 수

시계열 데이터  

시계열 데이터의 주요특성
- 시간순서: 일정한 시간 간격을 두고 기록하기 때문에 데이터 포인트의 순서가 중요  
- 자기상관성: 시간에 따라 연속적으로 기록하기 때문에 이전데이터와 현재 데이터간 포인트 사이 상관관계가 존재할 수 있음  
    ex) 구름이 끼기 때문에 나중에 비가 올 수 있다.    
- 추세: 장기적으로 증가하거나 감소하는 경향  
    ex) 주식  
- 계절성: 계절뿐만 아니라 일정한 주기로 반복되는 패턴을 의미  
    ex) 여름에는 여름과일이 잘 팔린다.  
- 변동성: 미시적으로는 변화가 일정하지 않고 불규칙적으로 변화할 수 있고,  
    과거데이터만으로 미래를 완벽하게 예측할 수 없기 때문에 변동성이 크게 나타날 수 있음  
<br>  

시계열 데이터 분석기법  
시각화  
- 시계열 데이터를 분석하기 전에 시각화를 통해 데이터의 전반적인 패턴, 추세, 계절성을 파악  
분해  
- 시계열 데이터를 추세(Trend//전체방향), 계절성(Seasonality//반복 패턴), 잔차(Residual//노이즈:통제 못하는 의미없는 랜덤한 흔들림)로 분해하여 각각의 성분을 분석  
이동 평균  
- 데이터를 평활화(들쭉날쭉한 노이즈를 줄여 부드러운 흐름만 보이게 하는 것)하여 노이즈를 제거하고 추세 파악  
- 간단한 이동 평균(Simple Moving Average//평균)과 가중 이동 평균(Weighted Moving Average//최근 값에 더 비중) 등
ARIMA 모델  
- 자기 회귀 적분 이동 평균(ARIMA, AutoRegressive Integrated Moving Average) 모델: 시계열 데이터 예측에 널리 사용되는 통계적 모델  
- 데이터의 자기 상관성을 이용하여 미래 값을 예측  
SARIMA 모델  
- 계절성을 고려한 ARIMA 모델로, 계절적 패턴이 있는 시계열 데이터를 분석하고 예측하는 데 사용  
지수 평활법  
- 최근 데이터에 더 큰 가중치를 부여하여 미래 값을 예측하는 방법  
- 단순 지수 평활법(Simple Exponential Smoothing)과 이중 지수 평활법(Double Exponential Smoothing) 등  
  
시계열 데이터의 활용  

경제 분석  
• GDP, 실업률, 물가 상승률 등의 경제 지표를 분석하여 경제 상태를 평가하고 예측합니다.  
• 예) 정부와 중앙은행은 GDP 예측을 통해 경제 정책을 결정하고, 기업은 경제 성장률을 기반으로 투자 결정을 내립니다.  
  
금융 시장

• 예) 투자자들은 주식의 매수 및 매도 시점을 결정하고, 포트폴리오를 관리

주의할 점  
  
인지편향  
주요 인지편향  
확증 편향 (Confirmation Bias)  
- 자신이 이미 가지고 있는 신념이나 가설을 확인하려는 경향  
- 새로운 증거나 데이터를 선택적으로 수용하는 현상  
ex) 지금 런칭할 서비스가 젊은층에 인기가 있을 것이라는 확신을 가지고 젊은 층을 대상으로만 긍정적인 피드백만 분석하게 될 경우  
다른 연령대의 데이터를 고려하지 않아 실제 전연령대에 제품이 인기가 있음을 간과할 수 있음  
- 예방 방법: 분석 과정에서 다양한 관점을 고려하고, 자신의 가설을 반증할 수 있는 증거도 적극적으로 탐색  
  
대표성 편향 (Representativeness Bias)  
- 어떤 사건이나 데이터를 특정 범주나 그룹의 대표로 간주하여 일반화하는 경향  
ex)한 투자자가 몇몇 성공 투자자가 중퇴를 한 특징을 알게되고 모든 대학 중퇴자가 성공할 것이라 생각해 이런 창업자에게만 집중해 다양한 배경을 가진 창업자는 눈여겨보지 않음  
- 예방방법: 표본이 충분히 크고 다양한지를 확인하며, 통계적으로 유의미한 결론을 도출할 때까지 충분한 검증을 거침  
  
후견 편향 (Hindsight Bias)  
- 일이 벌어진 후에 결과를 예측할 수 있었다고 믿는 경향  
ex) 한 기업이 새로운 제품을 출시하고 예상보다 큰 성공을 거두고 경영진은 처음부터 성공할 수 있었다고 주장할 때  
실제 제품 출시전 많은 불확실성이 있었고 성공은 결과론적인 판단에 불가  
- 예방 방법: 과거 데이터를 분석할 때 당시에 사용 가능한 정보와 조건만을 고려하고, 결과에 대한 후속 정보를 배제  
  
정박 편향 (Anchoring Bias)  
- 초기 정보에 지나치게 의존하여 이후의 판단에 영향을 미치는 현상  
ex) 한 부동산 중개인이 집 가격을 매우 높게 해서 구매자들이 높은 가격을 기준으로 협상해 실제가치보다 높은 가격으로 구매하게 되는 것  
- 예방 방법: 초기 가정이나 초기 데이터에 너무 의존하지 않고, 다양한 출처의 데이터를 종합적으로 분석  
  
가용성 편향 (Availability Bias)  
- 사람들이 어떤 사건의 발생 확률을 판단할 때, 쉽게 떠올릴 수 있는 정보나 예시를 기반으로 판단하는 경향  
ex) 한 뉴스 기사가 특정 범죄가 급증하고 있다고 보도할 때 이를 접한 사람들은 자신의 지역에서도 이런 범죄가 실제 적게 발생함에도 많이 발생한다고 생각하는 것  
- 예방 방법: 직관이나 기억에 의존하지 않고, 객관적이고 신뢰할 수 있는 데이터에 기반하여 판단  
  
자기충족적 예언 (Self-Fulfilling Prophecy)  
- 어떤 예언이나 기대가 그 자체로 사람들의 행동에 영향을 미쳐 예언된 결과를 실제로 일어나게 만드는 현상  
ex) 한 학생이 특정 과목에서 잘 못할거라는 선생님의 말을 듣고 제대로 공부를 안해서 결과론적으로 그런 결과가 나오게 되는 것  
- 예방 방법: 사람들을 평가할 때는 주관적인 예언이나 기대보다는 객관적인 기준과 데이터를 사용하여 평가
