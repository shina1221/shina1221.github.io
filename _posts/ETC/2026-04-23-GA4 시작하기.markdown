디지털분석: 우베사이트나 모바일 앱과 같은 디지털 소스에서 데이터를 수집, 측정 및 분석하는 과정  
aqcquisition(고객 확보): 서비스에 대한 인지도를 높이고 사용자들의 관심을 얻는 것  
어떤 마케팅 채널이 웹사이트나 앱으로 유도하는지 확인 가눙  
ex) 구글 상품 스토어를 어떻게 사용자가 찾는지, 어디서 앱을 찾는지   
engagement(참여): 사용자가 비즈니스와 상호작용 하는 것  
어떤 콘텐츠에 사용자들이 관심을 갖는지 다른 사람들과 공유하는지 확인 가능,어느 화면을 방문했는지 확인가능  
ex)어떤 제품을 보고 장바구니에 담는지, 어떤 화면에서 막히는지 
monetization(수익창출) and retention(고객유지): 사용자가 상품을 구매하는 것   
웹사이트나 앱에 얼마나 자주 사용자들이 재방문 하는지 측정 가능, 앱 내에서 얼마나 많은 사용자가 구매하는지 확인 가능    
</br>  
![데모](https://support.google.com/analytics/answer/6367342#zippy=%2C%EC%9D%B4-%EB%8F%84%EC%9B%80%EB%A7%90%EC%97%90%EC%84%9C%EB%8A%94-%EB%8B%A4%EC%9D%8C-%EB%82%B4%EC%9A%A9%EC%9D%84-%EB%8B%A4%EB%A3%B9%EB%8B%88%EB%8B%A4)  
</br>
account(계정) - 단일 사업체에 의해 소유되는 property들의 모음  
property(속성) - 웹사이트나 앱 같은 사용자 데이터 그룹을 나타내는 속성  
GA4가 데이터를 처리하는 곳이며 구글애즈와 같은 다른 제품과 연결을 설정할 수 있는 곳  
각 property에는 데이터 스트림이 존재  
datastream(데이터 스트림) - 웹사이트나 앱에서 해당 property로의 데이터 흐름을 나타내는 데이터로  
어떤 데이터를 수집할지 등을 제어가능  
웹 용 데이터 스트림과 모바일 앱용 데이터 스트림이 존재  
만약 웹과 앱 두가지 서비스를 판매할 경우 웹용 데이터 스트림, ios앱용 데이터 스트림, 안드로이드 앱용 데이터 스트림이 존재  
</br>
웹 사용자 여정 측정시 단일 웹 데이터 스트림을 사용하는 것을 추천  
도메인에 걸쳐있는 웹 여정에 대한 일관된 사용자 및 세션 보고를 위해선  
단일웹 데이터 스트림과 도메인 간 측정(measure)를 결합해 사용하는 것이 좋음  
</br>  
회사당 하나의 계정에 하나의 브랜드 혹은 하나의 사업부 당 하나의 property를 보유하는 계정구조를 설정하는 것이 좋음  
ex)만약 어느 배달회사가 고객용과 배달 기사용으로 구분된 두개의 웹사이트와 어플을 운영하고 있다면 
하나의 account, 두 개의 property(배달기사용, 고객용)로 이루어질 것이며  
각 property 당 하나의 웹 데이터 스트림, 안드로이드 앱 데이터 스트림, ios 앱 데이터 스트림으로 구성해 총 6개의 데이터 스트림을 보유할 것임  
</br>  
애널리틱스 360은 GA4의 기업용 버전  
</br>  
GA4로 가서 
account 생성 후 property 생성 시 business objectives를 선택하게 되는데  
여기서 선택한 비즈니스 목표에 맞는 보고서를 생성하게 됨  
만약 확실하지 않다면 get baseline reports 선택  
</br>  
웹사이트 데이터 스트림을 생성하기 위해선 URL과 Stream name을 설정하게 되는데 이러면 웹사이트에 특화된 웹사이트 태그가 생성됨  
웹사이트 빌더 혹은 CMS 시스템을 사용하는 경우 measurement ID를 복사해 웹사이트 빌더 계정에 붙여넣어야함  
웹사이트 빌더를 사용하지 않는 경우 웹사이트에 구글 태그를 직접 설치해야 함  
</br>  
**첫번째 방법은 구글 태그 관리자를 사용해 태그를 설치하는 것임**  
구글 태그 관리자를 사용하면 웹사이트 페이지에 코드를 설치하고 계정 인터페이스를 통해 언제든지 해당 코드를 추가하거나 변경가능. 따라서 이상한 코드를 수정하기 위해 돌아갈 필요 없음       
태그 관리자는 마케팅 및 웹사이트 태그를 관리하고 업데이트 하는 중앙 집중식 플랫폼 역할을 수행하게 됨  
따라서 구글 애널리틱스 태그 설치시 구글 태그 관리자를 사용하는것을 권장    
</br>  
이미 구글 태그 관리자를 사용하고 있다면 태그 관리자 계정에 태그를 추가하면 됨  
구글 태그 관리자는 CMS 또는 웹사이트 빌더와 함께 사용 가능  
**  
Wordpress, Wix, Squarespace와 같이 코딩이나 전문기술 없이도 웹사이트 콘텐츠를 쉽개 생성, 편징, 게시, 관리가능하게 해주는 SW(Contetnt Management System 이하 CMS)  
CMS는 기능 확장과 체계적인 콘첸츠 관리에 강해 대규모 사이트에 적합하고,  
웹사이트 빌더는 드래그앤 드롭 방식으로 코딩지식 없이 빠르고 간단하게 사이트 제작하는데 특화  
**
</br>  
따라서 웹사이트 빌더의 구글 애널리틱스 연동 기능에 측정 ID를 직접 입력하는 대신  
구글 태그 관리자에서 구글 애널리틱스 태그를 생성해야함  
</br>  
**두번째 방법으로 사이트의 각 페이지에 태그를 수동 설치하는 방법이 있음**  
이 옵션을 선택하면 UI에 새 웹사이트 태그가 표시됨  
**이 태그는 웹사이트의 모든 페이지에서 <head> 태그 바로 뒤에 설치해야함**  
웹사이트에 애널리틱스 태그를 설치하면 구글 애널리틱스 property로 다양한 데이터가 자동 수집됨  
더 구현을 진행하면 추천 이벤트와 사용자 지정 이벤트를 활용해 데이터 수집을 더욱 향상시킬 수 있음  
</br>  
앱 데이터 스트림은 firebase sw개발키트(SDK)를 기반으로 함  
이미 fire base를 사용하고 있다면 fire base 콘솔에서 분석기능 활성화  
만약 그렇지 않다면  
GA4의 관리자 페이지에서 데이터 스트림을 클릭해 생성  
안드로이드의 경우 패키지 이름을 필요로 하고, ios의 경우 번들 ID를 필요로 함  
GA4가 firebase 프로젝트를 생성하고 앱 데이터 스트림을 설정하기 시작할 것임  
</br>  
사용자를 측정하기 위해선 사용자 또는 앱 개발자가 앱에 firebase SDK를 구현해야함  
앱에 firebase를 설정하면 첫 오픈, 인앱 구매, 화면 조회 수 등 다양한 이벤트가 자동으로 수집됨  
또한 비즈니스와 관련된 추천 이벤트나 맞춤 이벤트를 수집할 수도 있음  
앱 코드에서 log event 메소드를 사용해 최대 500개의 서로 다른 이벤트 이름을 정의하고 수집 가능하며 총 이벤트 수에는 제한이 없음  
웹과 웹 데이터 수집 방식은 조금씩 다르지만 GA4에서 통합 분석가능  
다양한 기기에서 어떻게 탐색하고 ㅇ웹이나 앱과 어떻게 상호작용하며 가입, 로그인 구매같은 것을 어떻게 수행하는지 파악 가능  
SDK는 GA4에 연결하기 위해서가 아닌 사용자를 측정하기 위해 필요  
</br>  
구글 애널리틱스가 데이터 수신을 시작하면 데이터는 실시간 보고서에 나타나고 이후에 다른 보고서에 나타나서  
제대로 데이터가 수집되는지 확인하려면 실시간 보고서를 확인  
property에서 realtime항목으로 방문자 트래픽이 구글 애널리틱스에 나타나는지 확인  
데이터가 표시되지 않는다면 웹사이트나 앱을 열어 활성상태로 전환  
**실시간 보고서는 지난 30분동안 발생한 사용자 활동을 보여줌**  
사용자수와 사용중인 기기 목록 확인 가능  
데이터가 제대로 수집되고 있는지 확인하려면  
1. 지난 30분 동안 활동 내역이 있는지 2. 예상했던 데이터가 제대로 입력되고 있는지 확인 
ex)만약 이벤트 파라미터에서 보고자 하는 이벤트를 하나 클릭하고 상새 파라미터를 확인해서 페이지가 예상대로 나타나지 않는다면  
데이터 수집 과정에서 발생하는 오류를 수정하거나 수집하고 있는 데이터 종류를 확인해볼 것 
만약 제대로 값이 나타나지 않는다면 웹사이트 태그 구현에 문제가 있어서 그런것일 수 있음  
</br>  
관리자 메뉴는 account, property, data stream을 관리하는데 필수적인 도구  
맞춤 측정기준, 측정하목, 잠재고객같은 고급 기능까지 생성하고 관리  
직원이나 동료가 보고서, 데이터에 접근할 수 있도록 하려면 계정에서 acccount access managemnet를 통해 사용자를 추가하거나 삭제가능  
관리는 애널리틱스의 좌측 최하단 톱니바퀴 모양을 클릭하면 열 수 있음  
데이터 스트림 설정에서는 향상된 측정 설정을 수정하고 기본 태그 설정을 구성하는 등의 작업 수행 가능  
데이터 수집 가능을 활성화 비활성화 가능  
쿠키, 사용자 식별자 또는 광고 식별자와 관련된 수집데이터의 보존 기간 조정가능  
**이 설정은 집계된 데이터를 기반으로 하는 대부분의 표준 보고서에는 영향을 미치지 않으나 탐색 같은 고급 도구에서 사용되는 데이터에는 영향을 미침**  
생성한 계정이나 데이터 스르림등은 삭제 가능한데 삭제하면 trash can으로 보내져 35일동안 보관된 후 영구삭제 됨   
</br>  
이벤트와 매개변수가 전송되면 
이벤트는 사용자가 앱을 처음 열거나, 동영상을 시청하거나, 웹사이트의 페이지를 볼 때 발생할 수 있음  
이벤트는 이벤트 매개변수라고 하는 추가데이터와 함께 전송되며 이는 이벤트에 맥락을 추가함  
ex)누가 웹사이트 동영상 시청 시 동영상 제목, 시청시간 같은 이벤트 매개변수가 함께 전달됨    
이벤트 및 이벤트 매개변수와 함께 사용자의 지리적 위치, 사용중인 장치같은 사용자 데이터(사용자 속성)도 전송됨  
</br>  
**구글 애널리틱스는 매개변수, 이벤트 매개변수, 사용자 속성을 통해 데이터를 집계하고 보고서를 생성**  
**구글 애널리틱스는 시각적으로 표현할 때 차원과 측정항목을 사용**  
차원은 텍스트, 숫자 같은 데이터의 속성으로 기본값은 텍스트로 이벤트 이름(웹이나 앱에서 사용자가 발생시킨 이벤트의 이름)이 이에 속함  
측정값은 정량적 수치로 항상 숫자로 표시되며 텍스트로는 표시되지 않음 또한 수학적 연산 적용 가능  
구글 애널리틱스에는 페이지 조회수 및 구매같은 웹사이트 정보를 수집하는데 사용할 수 있는 여러 일반적인 이벤트와  
매개변수가 이미 포함되어 있지만  
**향상된 측정으로 수집되는 이벤트 같은 경우 보고서에 표시되도록 맞춤 측정기준과 측정항목을 등록해야함**
