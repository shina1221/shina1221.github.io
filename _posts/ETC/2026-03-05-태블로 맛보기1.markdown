프로젝트 모니터링 시 고려해야하는 점  
Key Perfirmance Index: 꾸준히 모니터링 하기 위함   
Business Alignment: 비즈니스 목표와 연결이 되어야 전략적 방향을 설정하고 목표를 달성하기 위한 방법을 식별하기 위함   
Decision Making:기업 내에서 액션 플랜으로 이어질 수 있는 계획들을 세우고 관리하기 위함      
  
기획/PM 직군의 대시보드 시각화  
1. 사용자 참여도 분석(바차트, 라인차트)  
-모바일 앱의 사용자 참여도를 분석해 주간활동 패턴, 월간 성장률 파악  
지표를 모니터링해 이커머스 매출이 급증하거나 사용자가 급증하는 기간에 앱의 주요업데이트나 프로모션, 특별 이벤트 기획  
2.기능 사용률 분석(히트맵)  
-불필요한 기능 및 직관적이지 않은 기능 확인  
서비스의 기능별 사용률을 확인하고 특정 기능을 자주 사용하지 않는다거나 자주 사용하는 부분에서 개선점을 파악하고 불필요한 서비스를 정리할 수 있고, 고객의 요구와 선호도 파악 가능  
어떤 페이지에서 자주 이탈하는지 어떤 기능을 자주사용하는지를 파악해 니즈에 맞는 서비스 기획 가능   
3. 사용자 유지율 분석(라인차트)  
-이탈 포인트 파악 및 유지를 위한 전략 제시  
신규 사용자가 앱 설치후 급격히 이탈하는 경향을 확인하거나 이런 트렌드를 발견해 어떻게 초기사용자들의 경험을 개선할 수 있는지 고민할 수 있음.  
4. A/B test 분석  
-데이터 기반 의사결정 지원  
5. 이상 징후 감지  
트래픽 감소나 오류율 증가를 조기에 감지  
특정 시간대에 트래픽이 감소하거나 오류율이 급증하는 것을 확인하고 대응 가능  
  
마케터를 위한 대시보드 시각화  
  
A/B 테스트란 
한 가지 변수(ex. 웹 사이트 디자인)을 A, B 두가지 버전으로 나눠서 테스트하며 어느 것이 더 효과적인지 판단하는 기법  
ex. 쇼핑몰 배너 디자인 (클릭, 구매 전환 테스트)  

1.목표 설정: 전환율이 높은 광고 배너를 찾는 것  
2.가설 설정: “바로가기”라는 문구를 “지금 구매하기”로 변경하면 더 많은 사용자가 클릭을 할 것이다.  
3.대상 군 분석  
1. Control Campaign(A): 기존의 문구 사용(바로가기)  
2. Test Campaign(B): 새로운 문구 사용(지금 구매하기)

검증 결과 significant test result에서 a안과 b안의 분포가 겹치지 않고 확연하게 다르다고 나옴  
즉 b안에 있는 통계 전환율이 유의미하게 달랐다고 해석  
*상대적 상승률(relative uplift): b가 A보다 전환율이 40% 개선  
*z-score: 표준화된 척도를 사용하여 각 데이터 값이 평균에서 얼마나 떨어져 있는지 측정하는 방법  
1.96	95% 신뢰  
2.58	99% 신뢰  
4.21	매우 높은 유의성  (결과값)  
  
*검정력(Statistical Power): 귀무가설(연구자가 부정하고자 하는 가설)이 실제로 거짓일 때,  
 이를 올바르게 기각하고 대립가설을 채택할 확률(통계적 검정의 능력)   
 제2종 오류(거짓 음성)를 범하지 않을 확률을 의미하며, 높을수록 연구의 신뢰도가 높음

A/B Test 멈춤 조건  
프로덕트를 배포하고 1주일간 데이터를 수집하여 A/B Test 결과를 분석했다가
미세한 차이가 있긴 했으나, 실험 결과 “통계적으로 유의한 차이가 아니다”는 결론이 나와  
계속 실험을 진행해야 할까?  미세한 차이가 있긴 하지만, 특정 안을 선택해도 괜찮을까? 를 고민하는 경우가 생길 것임  
  
A/B Test 적절한 주기 찾기  
• 추가 실험 진행: 더 많은 데이터 수집 (9:1 => 5:5)  
• 비즈니스 목표 및 전략 우선순위 고려 
- a안과 b안이 통계적으로 유의미한 차이가 없다면 비즈니스 전반적으로 생각하는 브랜드 이미지나 프로덕트의 장기적인 성능을 고려해 자체선택  
• A안과 B안의 운영 공수 및 비용 고려  
- 비용이 적게 들고 유지 보수 관리가 더 쉬운 옵션 선택  
• 사용자 경험(UX) 및 만족도 조사  
- 질적(정성적) 조사 결과나 사용자 피드백 참고하여 결정  
• 장기적인 전략과 Align  
- 기존안은 A안이고 변경된 안은 B안인데 비즈니스의 장기적인 전략과 맞게 주로 새로운 안 (B안)을 채택
