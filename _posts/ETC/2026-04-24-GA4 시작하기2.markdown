GA4에서 특정 기기에서 이벤트를 수집해 구글 애널리틱스 서버로 직접 보낼 수 있는 기능은 HTTP 요청  
</br>  
GA4에서 이벤트는 페이지 조회, 클릭 등 모든 사용자 행동을 수집하는 핵심 데이터 단위이며,  
세션은 이러한 이벤트가 모여 만들어지는 1회 방문 기록  
</br>  
측정기준(Dimension)은 데이터를 쪼개보는 '속성'이나 '분류 기준'(예: 기기, 국가, 페이지)이고,  
측정항목(Metric)은 합산된 '숫자 데이터'(예: 사용자 수, 조회수, 수익)  
</br>  
획득(acquisition)은 사용자가 어느 매체나 소스를 통해 들어왔는지를 맞춘 초점  
핵심 데이터: 첫 사용자 소스/매체, 세션 소스/매체, 신규 사용자 수, 세션 수  
참여(Engagement)는 사용자가 사이트나 앱 내에서 무엇을 하고(이벤트, 페이지 뷰 등)얼마나 머물렀는지에 초첨  
핵심 데이터: 참여 세션 수, 참여 시간, 참여율(Engagement Rate), 이벤트 수, 전환(Conversion)  
</br>  
만약 새로운 상품 페이지를 조회할 때 새로운 이벤트를 실행하도록 설정하려면? 이벤트 만들기  
관리>데이터표시>이벤트 혹은 맞춤정의로 이벤트 매개변수 생성(맞춤 측정기준 만들기)  
*소스 이벤트에서 매개변수 복사로 기존 이벤트(소스)가 가진 매개변수를 새로 만드는 이벤트에 그대로 가져옴  
이벤트 만들기 시 클릭한 버튼의 위치(link url 등) 같은 세부 데이터를 다시 설정할 필요 없음  
이벤트 네임 뷰 아이템에 추가하려는 이벤트값 추가  
</br>  
이벤트 수정 시 수정 이름이라는 행위에 대한 설명을 적어두고 수정하려는 매개변수 조건과 수정값을 넣으면 이후부터 의도한 대로 수집됨  
</br>  
이벤트를 중요 이벤트로 표시하기 위해 전환으로 표시 버튼을 눌러 주요이벤트로 표시  
</br>  
전환기여 분석을 하기위해 사용자를 식별하지 않고도 머신러닝을 이용해 전환을 예측할 수 있는 모델링된 전환 기능 존재  
사파리에서 접속시 쿠키는 7일만에 만료, 광고 파라미터가 있으면 24시간으로 줄어드는데  
이것 말고도 해외 사이트의 경우 쿠키에 동의하지 않는 경우도 있어서 머신러닝으로 어디서 전환이 일어났다를 예측해주는 것  
</br>  
구글 애즈 계정을 구글 애널리틱스 속성에 연결해 광고 데이터를 보려할 때  
관리 > 속성 > 구글애즈 링크 > 연결  
</br>  
구글 애널리틱스의 전환을 구글 애즈로 가져올 때 구글 애즈의 스마트 자동 입찰을 이용해 입찰가 최적화 가능   
</br>  
NPA(non_personalized_Ads)는 개인 맞춤이 아닌 광고로  
유럽연합에 거주하는 사용자에게 구글 애즈 광고를 하려는 기업이  
NPA가 1일 때 애널리틱스 데이터를 이용한 광고 불고  
0이라면 애널리틱스 데이터를 이용한 광고 가능(기본값)  
즉 개인 애널리틱스 데이터를 광고에 이용할 수 있는지 유무를 의미  
개인 맞춤 광고 사용 옵션은 관리의 하단에 있는 구글 애즈 링크 탭의 상세 링크를 통해 설정 가능  
NPA는 관리 > 데이터 > 맞춤정의를 통해 각 측정기준에 따라 설정 가능  
여기서 NPA표시가 되면 맞춤 광고에 사용하지 않으며 측정 목적으로만 사용되며 보고서 메뉴에서도 사용불가    
</br>  
예측 잠재고객 가눙은 관리>데이터 표시>구축을 통해 확인하며 구글애널리틱스에서 머신러닝을 기반으로 산출    
추천 잠재고객은 --할 가능성이 있는 사용자를 대상으로 함  
7일 이내에 구매할 가능성이 높은 사용자  
7일 이내에 앱을 제거할 가능성이 높은 사용자  
28일 동안의 예상 상위 송위자  
7일 이내에 첫 구매를 할 가능성이 높은 사용자  
7일 이내에 앱을 제거할 가능성이 높은 구매자를 대상으로 설정 가능  
각 항목은 예측 측정항목이라는 기준을 두어 기본요건 요건 충족시 사용 가능   
ex)최근 28일 동안 7일간의 기간에 대해 1천명 이상의 재사용자가 관련 예측조건(구매 또는 앱제거)를 트리거헀어야 하며  
다른 1천명 이상의 재사용장는 트리거하지 않았어야함  
서비스를 재사용중인 2천명(반반)에 대해 다음의 조건이 충족되어야 한다는 것  
따라서 사용자가 적다면 활성화가 불가할 것임  
</br>  
1달의 무료체험 후 프리미엄 회원으로 전환하는 사용자를 구분하기 위해  
회원 가입 후 14일 이내에 프리미엄 회원으로 전환하는 사용자를 구분하려면  
이 요건을 충족할 때마다 이벤트를 기록하려면 
구축(잠재고객)에서 새로운 잠재고객을 생성(맞춤 잠재고객)  
시퀀스를 추가해 조건을 생성 후 
잠재고객 트리거를 새로 생성해 이벤트 이름 추가(잠재구성원 여부가 변경될 때 추가 이벤트 로깅 체크)  
</br>  
서로 다른 기여 분석 모델이 유입 채널 평가에 미치는 영향을 비교하려할 때 광고메뉴의 모델 비교로 확인  
</br>  
전환 전에 발생한 첫번째 채널에 모든 전환 기여도를 부여하는것은 첫번째 클릭  
전환전에 발생한 모든 채널에 균등하게 전환 기여도를 배분하는 것은 선형 모드  
머신러닝 알고리즘을 사용해 다양한 터치포인트가 전환에 미치는 기여도를 배분하는 것이 데이터 기반 기여분석    
머신 러닝에 의해 전환에 미치는 기여도를 서로 다르게 봄  
</br>  
웹 사이트의 유입을 늘리기위해 검색광고, 소셜 광고등을 진행하고 있을 때  
이 광고들이 어떤 순서로 전환에 영향을 주는지 알기위해 전환경로를 알아보려 할때 전환 경로보고서는  
광고의 전환수에서 전환 기여분석을 통해 확인가능  
</br>  
전환에 근접합 시점에 발생한 터치포인트에 더 많은 기여도를 부여하는 기여모델은 시간가치 하락 모델  
마지막클릭은 마지막에 클릭한 소스에 기여도를 더 높게 평가하는 것  
</br>  
신규 사용자가 홈페이지를 방문한 다음 가장 많이 여는 페이지를 파악하기 위해 '탐색'에서 새 경로 탐색 분석을 만들었고 이를 동료와 공유할 계획입니다. 기본적으로 이 새 경로 탐색 분석을 볼 수 있는 사람은 누구인가요?
처음에는 본인만 접근 가능하나 공유시 다른 사용자에게 읽기 전용으로 제공됨  
기본 공유만으로는 편집 권한까지 자동으로 주어지지 않음
